Architecture Overview

AgentFlow reimagines Apache NiFi's dataflow paradigm as the foundation for composing, orchestrating, and governing autonomous AI agents. Each NiFi concept maps directly to an agentic AI primitive, providing enterprise-grade orchestration out of the box.

Interface Layer

3D React Three Fiber UI, NiFi Canvas, REST API Gateway, FlowSync (deployment & management library spanning all tiers).

Agent Orchestration

NiFi Runtime with Process Groups as agents, 7 custom processors, connections as communication channels.

Controller Services

Shared resources: LLM clients, vector databases, tool registries, guardrails enforcement.

Infrastructure

LLM APIs, Vector DBs, Kafka, Redis, Vault, ZooKeeper for clustering.

Tasks flow linearly through a chain of agents. Each agent enriches the FlowFile before passing it to the next. Ideal for workflows with a fixed sequence of stages.

👑 Supervisor Agent (Planner + Router)
↓ subtask ↑ result
Worker 1
↓ subtask ↑ result
Worker 2
↓ subtask ↑ result
Worker 3

A supervisor decomposes the task and delegates subtasks to specialized workers in parallel. Results are aggregated back to the supervisor. Ideal for tasks with independent sub-problems.

Agent A (Pro)
← argue → ← rebut →
Agent B (Con)
↓ both perspectives ↓
⚖ Judge Agent

Two agents debate from opposing perspectives. A judge agent synthesizes the arguments into a balanced consensus. Ideal for decisions requiring multi-perspective analysis.

✂ Splitter
Agent (chunk 1)
Agent (chunk 2)
Agent (chunk 3)
Agent (chunk N)
📦 Aggregator

A splitter breaks the task into N independent chunks, each processed by a separate agent in parallel. An aggregator merges all results. Ideal for large-scale data processing.

FlowFile Structure

The FlowFile is the unit of work flowing between agents. It carries the full task context as attributes (metadata) and content (JSON payload).

ATTRIBUTES (metadata):
task.id = "uuid-1234"
task.type = "research_query"
task.priority = "high"
task.origin_agent = "supervisor"
task.target_agent = "researcher"
task.iteration = 3
task.max_iter = 10
task.token_budget = 50000
task.tokens_used = 12340
task.status = "in_progress"
llm.model.id = "claude-sonnet-4-6"
llm.tokens.input = 2105
llm.tokens.output = 1136
CONTENT (JSON payload):
{
  "goal": "Research recent trends...",
  "conversation_history": [
    {"role": "user", ...},
    {"role": "assistant", ...},
    {"role": "tool", ...}
  ],
  "plan": {"steps": [...], "current_step": 2},
  "tool_results": [...],
  "memory_context": [...],
  "final_output": null
}
